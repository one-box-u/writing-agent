# 常见问题 FAQ

## 基础使用

### Q1: 如何开始第一篇文章？

直接对 Claude 说："帮我写一篇关于XXX的文章"，系统会自动引导你完成整个流程。

### Q2: 必须按照完整流程走吗？能不能跳过某些步骤？

**不能跳过核心步骤**。这是设计理念：
- 必须经过"澄清写作需求"
- 必须选择是否调研
- 必须经过"主编审稿"

但"调研"和"风格建模"是可选的，你可以选择跳过。

### Q3: 我不想每次都调研，能设置默认跳过吗？

不能设置默认，但每次会询问你"A.调研 or B.直接写作"，选B即可跳过。这是为了确保你有意识地做出选择。

## 风格建模

### Q4: 如何创建自己的风格？

1. 准备一篇你喜欢的文章（3000字以上效果更好）
2. 对 Claude 说："帮我分析这篇文章的风格"
3. 系统会提取12个维度的特征并保存到 `.claude/styles/`

### Q5: 风格文件可以手动编辑吗？

可以！风格文件就是普通的 Markdown 文件，你可以：
- 修改"禁用词汇"列表
- 调整"招牌动作"
- 补充"段落模板"

### Q6: 一个风格文件可以用于多篇文章吗？

当然可以！这就是风格建模的意义。创建一次，永久复用。

## 字数控制

### Q7: 为什么字数还是不准？

从 v1.6.0 开始，系统要求使用外部工具统计字数。如果还是不准，检查：
1. 是否在关键节点（开头/主体50%/主体80%）进行了检查？
2. 是否使用了 `write_to_file` + `run_command` 统计？

### Q8: 能不能调整字数允许范围？

可以。编辑 `.claude/skills/写作执行/SKILL.md`，找到"允许范围"表格，修改百分比（默认±20%）。

### Q9: 字数超标了怎么办？

系统会在红线（110%）时自动停止。你可以：
- 让 Claude 删减内容
- 或者接受超标，手动编辑文件

## 反AI味道

### Q10: 什么是"AI味道"？

典型特征：
- 小标题病（每隔两三段就来个"## 小标题"）
- 排比上瘾（"首先...其次...最后..."）
- 格式洁癖（段落长度高度一致）
- 假大空结尾（"让我们共同期待..."）

### Q11: 我就是想用小标题，怎么办？

在澄清需求时明确说："这篇文章需要用小标题"，系统会例外处理。

### Q12: 如何判断文章有没有AI味？

主编审稿环节会自动检测。你也可以自查：
- 读起来是否像教科书？
- 是否有大量"首先/其次"？
- 段落长度是否整齐划一？

## 文件管理

### Q13: 文章保存在哪里？

`articles/` 目录下，格式：`[标题]_[版本号]稿.md`

### Q14: 如何查看历史版本？

所有版本都保存了：
- `标题_1稿.md`（初稿）
- `标题_2稿.md`（修订稿）
- `标题_最终稿.md`（最终版）

### Q15: 能不能自动备份到云端？

项目本身不提供云备份，但你可以：
- 把整个项目放在 OneDrive/Google Drive 同步目录
- 或者定期 `git push` 到 GitHub 私有仓库

## 高级问题

### Q16: 如何添加新的 Skill？

1. 在 `.claude/skills/` 下创建新目录
2. 创建 `SKILL.md` 文件
3. 参考现有 Skill 的格式编写

详见 [Claude Code Skills 文档](https://code.claude.com/docs/skills)

### Q17: 能不能用于英文写作？

理论上可以，但：
- 反AI规则主要针对中文AI写作习惯
- 风格建模对中文效果更好
- 字数统计对英文也准确

建议针对英文单独调整规则。

### Q18: 这个系统适合写什么类型的文章？

适合：
- 观点文、随笔
- 深度分析文章
- 个人博客文章

不太适合：
- 学术论文（需要严格格式）
- 新闻稿（需要特定结构）
- 技术文档（需要大量小标题和列表）

## 故障排除

### Q19: Claude 没有按照工作流执行怎么办？

检查 `CLAUDE.MD` 文件是否在项目根目录，并且内容完整。

### Q20: 风格库检查时提示"为空"，但我明明有风格文件？

检查：
1. 风格文件是否在 `.claude/styles/` 目录下？
2. 文件扩展名是否为 `.md`？
3. 目录路径是否正确？

### Q21: 字数统计命令报错？

Windows 用户确保使用 PowerShell（不是 CMD）：
```powershell
(Get-Content articles/temp_count.txt -Raw).Length
```
