# 选题库 (Topic Pool)

这个文件夹用于存储 AI 生成的候选选题，即使暂时不写，也可以保存下来供日后参考。

## 📁 文件结构

每个选题文件的命名格式：`YYYY-MM-DD-选题名称.md`

例如：
- `2026-01-15-职场35岁危机实操指南.md`
- `2026-01-16-技术转管理的5个坑.md`

## 📝 文件内容

每个选题文件包含：
- **选题名称**：简洁明确的标题
- **综合评分**：AI 评估的分数（满分 100）
- **子话题拆解**：可以独立成文的 3-5 个子话题
- **开头钩子**：为每个子话题设计的候选开头
- **素材清单**：需要调研的数据、案例、观点
- **生成日期**：记录选题的时效性

## 🔄 使用流程

### 1. 生成选题
当你对 AI 说"帮我想个选题"时，AI 会使用 `topic-generator` 技能生成候选选题。

### 2. 保存到选题库
如果暂时不写，可以说"保存到选题库"，AI 会自动创建文件并保存到这个文件夹。

### 3. 定期回顾
建议每月回顾一次选题库：
- 检查哪些选题的热度上升了
- 删除已经过时的选题
- 将值得写的选题移到写作计划中

### 4. 重新验证
对于超过 3 个月的选题，建议重新使用 `topic-research` 验证热度和竞争情况。

## 📊 选题状态管理

在每个选题文件的开头，可以手动标记状态：

- `待写作`：已生成，等待开始
- `调研中`：正在收集素材
- `写作中`：已经开始写作
- `已完成`：文章已发布
- `已过时`：话题热度已过，不建议再写

## 💡 最佳实践

1. **不要囤积太多**：选题库不是越多越好，保持 10-20 个高质量选题即可
2. **优先写高分选题**：综合评分 80 分以上的选题优先级最高
3. **关注时效性**：热点型选题要尽快写，常青型选题可以慢慢打磨
4. **定期清理**：每月删除评分低于 60 分或已过时的选题

## 🎯 快速查找

可以使用以下命令快速查找选题：

```powershell
# 查看所有选题
Get-ChildItem -Path "articles\_topic_pool" -Filter "*.md"

# 按日期排序（最新的在前）
Get-ChildItem -Path "articles\_topic_pool" -Filter "*.md" | Sort-Object LastWriteTime -Descending

# 搜索包含特定关键词的选题
Get-ChildItem -Path "articles\_topic_pool" -Filter "*职场*.md"
```
