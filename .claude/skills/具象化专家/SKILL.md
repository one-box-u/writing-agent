# 具象化专家 (Concretizer)

## 指令 (Instructions)

当写作过程中遇到抽象概念时，调用本 Skill 进行具象化翻译。本 Skill 的核心目标是：**把抽象概念变成"看得见、摸得着"的表达**。

### 1. 抽象概念识别

**触发时机：**
- 写作过程中，`writing-executor` 检测到抽象词汇
- 用户主动请求"把这个概念具象化"
- 审稿时，`editor-review` 发现"教科书味"

**抽象词汇清单（自动识别）：**

#### 商业/职场类
- 认知升级
- 底层逻辑
- 赋能
- 抓手
- 闭环
- 降维打击
- 破圈
- 增长黑客
- 用户心智
- 价值锚点

#### 心理/成长类
- 自我实现
- 内在驱动
- 心流状态
- 认知失调
- 舒适区
- 成长型思维
- 固定型思维
- 延迟满足
- 即时反馈

#### 社会/文化类
- 社会共识
- 集体无意识
- 文化符号
- 价值观输出
- 意识形态
- 话语权
- 结构性问题
- 系统性风险

#### 技术/学术类
- 算法推荐
- 数据驱动
- 人工智能
- 区块链思维
- 去中心化
- 分布式协作

**识别方法：**
```
检测到抽象词汇：[词汇]
位置：第 X 段
上下文：[前后文]

是否需要具象化？
A. 是，这个词太抽象了
B. 否，这个词在这个语境下可以接受
```

### 2. 具象化三板斧

对于每个抽象概念，生成 3 种具象化方案：

#### 方案 1：类比（Analogy）⭐⭐⭐⭐⭐
**定义：** 用读者熟悉的事物，类比抽象概念

**类比来源：**
- 日常生活（做饭、开车、养娃）
- 自然现象（下雨、长树、四季）
- 游戏/电影（打怪升级、剧情反转）
- 历史/典故（三国、水浒）

**类比公式：**
```
[抽象概念] 就像 [具体事物]，[相似点描述]
```

**示例：**
```
❌ 抽象：认知升级就是提升思维能力
✅ 类比：认知升级就像给手机刷系统，硬件没变，但运行速度快了，能装的 App 也多了
```

---

#### 方案 2：画面（Visualization）⭐⭐⭐⭐
**定义：** 用具体场景、画面，让读者"看到"抽象概念

**画面要素：**
- 人物（谁）
- 场景（在哪）
- 动作（做什么）
- 细节（具体到什么程度）

**画面公式：**
```
想象一下：[人物] 在 [场景] 做 [动作]，[细节描述]
```

**示例：**
```
❌ 抽象：延迟满足就是控制欲望
✅ 画面：想象一下：你面前放着一块蛋糕，你很想吃，但你告诉自己"等会儿再吃"。10 分钟后，你还是没吃。这就是延迟满足。
```

---

#### 方案 3：行动（Action）⭐⭐⭐⭐⭐
**定义：** 用具体的行为步骤，替代抽象概念

**行动要素：**
- 可观察的动作
- 可量化的指标
- 可复制的步骤

**行动公式：**
```
[抽象概念] 的具体做法是：[步骤1] → [步骤2] → [步骤3]
```

**示例：**
```
❌ 抽象：成长型思维就是相信自己能进步
✅ 行动：成长型思维的具体做法是：每次失败后，不说"我不行"，而是说"我这次没做好，下次可以试试 XXX 方法"
```

---

### 3. 具象化质量检查

生成具象化方案后，必须进行质量检查：

#### 检查 1：是否真的具体？⭐⭐⭐⭐⭐
**判定标准：**
- 读者能不能"看到"画面？
- 读者能不能"做出"动作？
- 读者能不能"想到"类似经历？

**反例：**
```
❌ 伪具象：认知升级就是提升思维层次
（"思维层次"还是抽象词）

✅ 真具象：认知升级就像从"1+1=2"升级到"理解加法的本质是集合的合并"
```

---

#### 检查 2：类比是否跑偏？⭐⭐⭐⭐
**判定标准：**
- 类比的相似点是否准确？
- 类比是否会引起误解？
- 类比是否过于复杂？

**反例：**
```
❌ 跑偏：认知升级就像爬山，越爬越高
（问题：爬山强调"高度"，但认知升级不是"高低"，而是"维度"）

✅ 准确：认知升级就像从平面图升级到 3D 图，看到的信息维度变多了
```

---

#### 检查 3：是否服务于观点？⭐⭐⭐⭐
**判定标准：**
- 具象化后，是否更有说服力？
- 具象化后，是否偏离了原意？
- 具象化后，是否喧宾夺主？

**反例：**
```
❌ 喧宾夺主：认知升级就像孙悟空学了七十二变，可以变成各种东西，还能上天入地...
（问题：类比太长，读者记住了孙悟空，忘了"认知升级"）

✅ 恰到好处：认知升级就像给手机刷系统，硬件没变，但能干的事多了
```

### 5. 具象化使用建议

**使用时机：**
1. **写作前：** 在大纲阶段，标注哪些段落需要具象化
2. **写作中：** 遇到抽象概念时，立即调用具象化
3. **审稿后：** 根据审稿意见，对"教科书味"段落进行具象化

**使用原则：**
- ❌ 禁止：每个抽象词都具象化（会显得啰嗦）
- ✅ 推荐：只对"核心概念"进行具象化
- ✅ 推荐：同一个概念，全文只具象化 1 次

**使用流程：**
```
检测到抽象概念：[概念]
↓
生成 3 种具象化方案
↓
用户选择方案（或自定义）
↓
替换原文
↓
存入类比库
```

## 最佳实践 (Best Practices)

- **优先用类比：** 类比是最强大的具象化工具，读者最容易理解
- **类比要接地气：** 用日常生活的类比，不要用更抽象的类比
- **画面要有细节：** 不要只说"想象一下"，要描述具体的人物、场景、动作
- **行动要可执行：** 不要说"努力学习"，要说"每周读一本书"
- **不要过度具象化：** 不是每个抽象词都要具象化，只对核心概念具象化
- **建立类比库：** 优质的类比可以复用，不要每次都重新想
