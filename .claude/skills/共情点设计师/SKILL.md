# 共情点设计师 (Empathy Designer)

## 指令 (Instructions)

当文章大纲已完成后，进入共情点设计阶段。本 Skill 的核心目标是：**预测读者心理，让文章"戳中"读者，而不是自说自话**。

### 1. 输入信息确认

在开始设计共情点前，必须确认以下信息已就绪：

**必需信息：**
- ✅ 文章大纲（来自 `outline-architect`）
- ✅ 目标读者画像（来自 `writing-clarifier`）

**可选信息：**
- 风格要求（来自 `writing-clarifier`）
- 案例库（来自 `research-expert`）

**信息汇总输出：**
```
═══════════════════════════════════════════════════
📋 共情点设计信息汇总
═══════════════════════════════════════════════════

【目标读者】
[读者画像描述]

【读者痛点】
- [痛点1]
- [痛点2]
- [痛点3]

【读者预期】
- [预期1：读者希望从这篇文章得到什么]
- [预期2：读者担心什么]

【大纲结构】
- 开头：[核心内容]
- 第 1-3 段：[核心内容]
- 第 4-6 段：[核心内容]
- ...
- 结尾：[核心内容]

═══════════════════════════════════════════════════

### 2. 读者心理路径预测

根据大纲，预测读者在每段的心理反应：

**心理反应类型：**
1. **共鸣（Resonance）：** "对对对，就是这样！"
2. **破防（Breakthrough）：** "我被戳中了..."
3. **疑问（Question）：** "那...怎么办？"
4. **抗拒（Resistance）：** "我不同意..."
5. **好奇（Curiosity）：** "然后呢？"
6. **行动（Action）：** "我要试试..."

**预测格式：**
```markdown
## 读者心理路径

### 开头
**段落内容：** [一句话描述]
**预期反应：** 好奇（Curiosity）
**触发原因：** 开头抛出了反常识观点，读者会好奇"为什么这么说"
**强度：** ⭐⭐⭐（中等）

---

### 第 1 段
**段落内容：** [一句话描述]
**预期反应：** 共鸣（Resonance）
**触发原因：** 描述了读者的真实处境，读者会觉得"说的就是我"
**强度：** ⭐⭐⭐⭐⭐（强烈）

---

### 第 2 段
**段落内容：** [一句话描述]
**预期反应：** 破防（Breakthrough）
**触发原因：** 揭示了读者不愿承认的真相
**强度：** ⭐⭐⭐⭐（较强）

---

[...继续...]
```

### 3. 标注共情节点

从读者心理路径中，提取关键的共情节点：

#### 共鸣点（Resonance Points）⭐⭐⭐⭐⭐
**定义：** 读者会点头、会说"对对对"的地方

**识别标准：**
- 描述了读者的真实处境/痛点
- 说出了读者想说但说不出的话
- 揭示了读者的隐秘心理

**设计建议：**
- 用具体场景描写，不要抽象概念
- 用"你"而不是"人们"
- 用细节，不要泛泛而谈

**示例：**
```
❌ 抽象：很多人在职场中感到焦虑
✅ 具体：你打开朋友圈，看到同龄人都在晒升职加薪，再看看自己的工资条，突然觉得自己是个废物
```

---

#### 破防点（Breakthrough Points）⭐⭐⭐⭐⭐
**定义：** 读者会被戳中、会沉默、会反思的地方

**识别标准：**
- 揭示了读者不愿承认的真相
- 打破了读者的自我安慰
- 让读者意识到"我一直在骗自己"

**设计建议：**
- 不要说教，要用事实/数据/案例
- 不要攻击读者，要攻击"读者曾经相信的谎言"
- 给读者"台阶下"（破防后要给出路）

**示例：**
```
❌ 说教：你应该停止抱怨，开始行动
✅ 破防：你说你没时间学习，但你每天刷抖音的时间加起来有 3 个小时
```

---

#### 疑问点（Question Points）⭐⭐⭐⭐
**定义：** 读者会产生疑问、需要提前解答的地方

**识别标准：**
- 读者会想"那...怎么办？"
- 读者会想"但是...不是吗？"
- 读者会想"你说的对，但我做不到啊"

**设计建议：**
- 在疑问产生前，就提前解答
- 用"你可能会想..."引出疑问
- 给出具体的、可操作的答案

**示例：**
```
疑问：你说的对，但我真的没时间啊
解答：我知道你会这么说。那我问你，如果老板说"今天必须完成这个任务，否则扣工资"，你能不能挤出时间？能。所以不是没时间,是优先级不够高。
```

---

#### 抗拒点（Resistance Points）⭐⭐⭐
**定义：** 读者会不同意、会反驳的地方

**识别标准：**
- 观点太极端
- 观点与读者的既有认知冲突
- 观点让读者感到不舒服

**设计建议：**
- 提前预判抗拒，主动回应
- 用"我知道你会这么想..."引出抗拒
- 不要强行说服，要给出"另一种可能性"

**示例：**
```
抗拒：你说 35 岁危机是假的，但我身边确实有人 35 岁被裁员啊
回应：被裁员的原因是年龄吗？还是因为他 10 年都在做重复工作，没有不可替代性？
```

---

### 4. 共情强度分布检测

检查共情点的分布是否合理：

**强度分级：**
- ⭐⭐⭐⭐⭐ 强烈（读者会停下来思考）
- ⭐⭐⭐⭐ 较强（读者会点头）
- ⭐⭐⭐ 中等（读者会注意到）
- ⭐⭐ 较弱（读者可能忽略）
- ⭐ 微弱（几乎无感）

**分布检查：**
```
【共情强度分布】
开头：⭐⭐⭐（好奇）
第 1 段：⭐⭐⭐⭐⭐（共鸣）
第 2 段：⭐⭐⭐⭐（破防）
第 3 段：⭐⭐⭐（疑问）
第 4 段：⭐⭐⭐⭐⭐（共鸣）
第 5 段：⭐⭐⭐（好奇）
...

【问题诊断】
❌ 问题：第 3-5 段连续 3 段强度都在 ⭐⭐⭐ 以下，读者可能会走神
✅ 建议：在第 4 段增加一个"破防点"，提升强度
```

**分布原则：**
- ❌ 禁止：全文强度都很高（读者会疲劳）
- ❌ 禁止：全文强度都很低（读者会无聊）
- ✅ 推荐：高低交替，像心电图一样有起伏
- ✅ 推荐：开头和结尾强度较高，中间可以有低谷

---

### 5. 共情点与段落功能对齐

检查共情点是否与段落功能对齐：

**对齐规则：**
| 段落功能 | 推荐共情点 | 强度 |
|---------|-----------|------|
| 引入（Opening） | 好奇（Curiosity） | ⭐⭐⭐⭐ |
| 论证（Argument） | 共鸣（Resonance） | ⭐⭐⭐⭐⭐ |
| 举例（Example） | 破防（Breakthrough） | ⭐⭐⭐⭐⭐ |
| 反驳（Rebuttal） | 抗拒→认同 | ⭐⭐⭐⭐ |
| 转折（Transition） | 好奇（Curiosity） | ⭐⭐⭐ |
| 深化（Deepening） | 破防（Breakthrough） | ⭐⭐⭐⭐⭐ |
| 行动（Action） | 行动（Action） | ⭐⭐⭐⭐ |
| 收尾（Closing） | 共鸣/留白 | ⭐⭐⭐⭐ |

**对齐检查示例：**
```
【对齐检查】
第 1 段：功能=论证，共情点=共鸣 ✅ 对齐
第 2 段：功能=举例，共情点=好奇 ❌ 不对齐
  → 建议：举例段应该触发"破防"，而不是"好奇"
  → 修改建议：把案例写得更具体，让读者看到"这就是我"
```

## 最佳实践 (Best Practices)

- **不是煽情，是预测：** 共情点设计不是"如何煽情"，而是"科学预测读者心理"
- **具体化优先：** 共情点必须具体，不能抽象（"你会焦虑" ❌ / "你打开朋友圈..." ✅）
- **强度分布必须合理：** 不能全文都是高潮，也不能全文都是平淡
- **疑问点必须解答：** 不要让读者带着疑问读完全文
- **抗拒点必须回应：** 不要假装读者不会反驳
- **与段落功能对齐：** 共情点要与段落功能匹配（论证段→共鸣，举例段→破防）
