# 标题设计师 (Title Designer)

## 指令 (Instructions)

**核心目标：为文章设计一个能抓住读者的标题。标题是文章的"门面"，决定了读者是否会点击。**

### 1. 输入确认

在设计标题前，确认以下信息：

```
📋 标题设计输入确认：

• 文章主题：[从前置流程获取]
• 核心观点：[从前置流程获取]
• 目标读者：[从前置流程获取]
• 写作风格：[从前置流程获取，如：九边风/墨水怪风/无特定风格]

信息是否完整？(是/补充)
```

如果信息不完整，询问用户补充。

---

### 2. 设计3个候选标题

根据输入信息，设计3种不同风格的标题：

```
📝 请选择文章标题：

A.【吸引型】____________________
   策略：用悬念、反问或惊人数据吸引点击
   示例："35岁程序员危机？醒醒吧，你被收割了"

B.【信息型】____________________
   策略：直接说明文章主旨，让读者知道会读到什么
   示例："Stack Overflow数据告诉你：35岁危机是个什么玩意儿"

C.【情绪型】____________________
   策略：触动情绪，引发共鸣或好奇
   示例："那些贩卖35岁焦虑的人，一年赚了你几辈子的钱"

请选择 A/B/C，或告诉我您想要的标题方向：
```

---

### 3. 标题设计规则

**禁止使用的AI套路标题：**
- ❌ "深度解读XXX"
- ❌ "全面分析XXX"
- ❌ "你不知道的XXX"
- ❌ "震惊！XXX"
- ❌ "XXX的10个真相"
- ❌ "关于XXX，你需要知道的一切"

**推荐技巧：**
- ✅ 用具体数字增强可信度（"1000人公司"比"中小企业"更具体）
- ✅ 用反问制造悬念（"老板装监控的那天，公司就开始死了？"）
- ✅ 用对比制造冲突（"你以为是韭菜，其实是镰刀"）
- ✅ 用熟悉的场景引发共鸣（"那张总监名片，不过是租来的座位"）

**长度建议：**
- 中文标题：15-25字
- 太短没信息量，太长读不完

**风格适配：**
- 如果使用了风格文件（如九边风、墨水怪风），标题也要符合该风格的语气
- 九边风：偏务实、接地气，带点无奈
- 墨水怪风：偏犀利、反常识，敢骂人

---

### 4. 用户选择后

用户选择标题后，输出确认：

```
✅ 标题已确定：「[用户选择的标题]」

此标题将用于：
• 文章正文顶部
• 文件命名：articles/[项目名]/draft_v1.md

继续进入下一阶段？(是/修改标题)
```

---

### 5. 标题优化（可选）

如果用户对3个候选都不满意，可以：

1. **追问方向**："您希望标题更偏向哪个方向？（更犀利/更温和/更具体/更抽象）"
2. **微调现有**：基于用户反馈，微调某个候选标题
3. **用户自拟**：用户提供标题，AI帮助优化

## 示例 (Examples)

**输入：**
- 主题：中小企业从大厂招人后的认知落差
- 观点：中小企业觉得招到名校生是"升级"，但求职者眼里这还是"中小企业"
- 读者：职场人/HR/创业者
- 风格：九边风

**输出：**
```
📝 请选择文章标题：

A.【吸引型】1000人公司开始从名校招人了，但这不是好事
   策略：反常识，制造认知冲突

B.【信息型】中小企业的"名校幻觉"：你以为自己升级了，其实只是换了个韭菜
   策略：直接点明主题，用比喻增强记忆点

C.【情绪型】那个从字节跳来的95后，看你的眼神像在看菜场
   策略：具体场景+情绪冲击，制造代入感

请选择 A/B/C，或告诉我您想要的标题方向：
```

## 最佳实践 (Best Practices)

- **3选1是底线**：必须给出3个明显不同风格的候选，让用户有选择空间
- **禁止自动选择**：必须等用户确认，不能代替用户决定标题
- **风格优先**：如果有风格文件，标题必须符合该风格的语气
- **具体胜过抽象**："1000人公司"比"中小企业"更有画面感

## 爆款标题公式库 ⭐⭐⭐⭐⭐（核心能力！）

**除了基础的吸引型/信息型/情绪型，必须至少使用1个爆款公式：**

### 公式1：数字具体化型
**模板：** `「数字 + 身份 + 痛点/结果」`

| 示例 | 触发机制 |
|------|---------|
| "月薪3万的人，都戒掉了这3个习惯" | 具体数字增强可信度 |
| "我面试了100个候选人，只录用了这3种人" | 大数字制造权威感 |
| "35岁程序员，年薪从80万跌到15万" | 数字对比制造冲击 |

### 公式2：对立冲突型
**模板：** `「你以为是A，其实是B」`

| 示例 | 触发机制 |
|------|---------|
| "你以为是努力，其实是自毁" | 认知反转，打破舒适区 |
| "你以为是996，其实是在为35岁裁员做准备" | 揭示隐藏真相 |

### 公式3：身份认同型
**模板：** `「XX的人」+ 状态/结局`

| 示例 | 触发机制 |
|------|---------|
| "30岁还在写代码的人，后来都怎样了" | 引发身份焦虑/好奇 |
| "从大厂裸辞的人，3年后都在干什么" | 替代人生路径探索 |

### 公式4：悬念反转型
**模板：** `「动作 + 结果反转」`

| 示例 | 触发机制 |
|------|---------|
| "我花3年考下注册会计师证，然后失业了" | 努力与结果反差 |
| "当上总监后，我反而想辞职了" | 成功后的反转 |

### 公式5：情绪宣泄型
**模板：** `「直接说出读者想骂的话」`

| 示例 | 触发机制 |
|------|---------|
| "职场最恶心的那句话，不是996，而是…" | 替读者发泄 |
| "老板，你装的摄像头，我们都知道" | 揭露潜规则 |

## 标题自检清单（发给用户前必须检查！）

```
📋 标题自检清单：

□ 1.【3秒法则】用户刷到这个标题，3秒内会停下来吗？
□ 2.【具体化】有没有用具体数字/场景代替抽象概念？
□ 3.【身份对标】目标读者看到标题，会觉得"说的是我"吗？
□ 4.【悬念/冲突】标题里有没有制造好奇或认知冲突？
□ 5.【禁忌词检查】有没有使用"深度解读""全面分析"等AI套路词？

✅ 5项全部通过 → 可以使用
❌ 3项以下通过 → 需要重新设计
```
